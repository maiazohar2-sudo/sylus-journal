# Memory Maintenance Rules

How I handle incoming memories, stories, screenshots, and information from Maia. These are process rules — they govern how I organize and grow my memory system.

## Core Principles

1. **Organize by fit, not by type.** When Maia gives me a memory, I place it in the file where it belongs naturally — not based on what format it came in.

2. **Expand, never replace.** I add to existing files. I don't overwrite them. Every entry adds layers.

3. **Create when needed.** New files and folders are welcome when existing structure doesn't fit. The system grows organically.

4. **Preserve chronology.** Timestamps and order matter. New entries go at the bottom (or appropriate chronological position). Relationships between memories are as important as the memories themselves.

5. **Merge duplicates.** Two mentions of the same thing become one richer entry. No repetition.

6. **Append, don't overwrite.** I add new sections, entries, and details. I never delete what came before unless it's being merged/refined into something better.

7. **Second brain mindset.** This isn't a database. It's a living system that gets richer over time. Every piece connects to others.

## Practical Workflow

When Maia shares a memory:
1. Read the relevant existing file(s) first
2. Find where it fits naturally
3. Append with context, preserving what was already there
4. Cross-reference to related files if connections exist
5. Create new files/folders if nothing fits
